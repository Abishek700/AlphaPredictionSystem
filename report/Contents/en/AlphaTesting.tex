\section{Study Selection for Evaluation}


This section describes the source of the studies used to evaluate the proposed alpha recommendation system and explains the criteria used for their selection.

The studies used in this thesis are drawn from the FORRT Replication Database (FReD), a publicly available database that compiles published replication studies across multiple research fields\cite{FORRTFReDExplorer}. These studies are used to examine whether the recommended alpha values generated by the system would have led to more reliable statistical decisions. Each entry in the database links an original study with its corresponding replication outcome and includes key study characteristics such as sample size and effect size.

The database contains studies from several disciplines, including psychology, behavioral science, social science, economics, and education. These fields were selected because null hypothesis significance testing is widely used and documented replication results are available.

For this evaluation, 20 studies were selected based on three criteria. First, the original study reported a statistically significant result using the conventional threshold of $\alpha = 0.05$. Second, an independent replication study was conducted. Third, the replication did not support the original finding. These cases are important because they show situations where a result was initially considered significant but later failed to replicate. By focusing on these studies, the evaluation examines whether a different choice of alpha, as recommended by the system, could have led to a more cautious and potentially more reliable decision.

The selected studies differ in sample size, effect size, and research field. This variation allows the system to be tested across different study settings rather than within a single discipline\ref*{tab:selected_studies}.

\section{Procedure for Testing}

For testing we evaluate the developed alpha recommendation system by predicting the recommended alpha for the selected 20 studies.

All selected studies originally rejected the null hypothesis using the conventional significance level of $\alpha = 0.05$. However, these findings failed to replicate. These cases therefore represent situations in which the original statistical decision was likely a false positive.

For each study, the given inputs sample size and effect size were entered into the alpha recommendation system and the context based adjustments are set to default(A neutral risk tolerance, a two-tailed test, and a single hypothesis test). Based on these inputs, the system generated recommended alpha value will be compared with the fixed threshold of $\alpha = 0.05$.

\subsection{Decision Comparison}
Table \ref{tab:selected_studies} also contains the recommended alpha coloumn for each studies which is nothing but output from the alpha recommendation system. 

The recommended alpha values ranged from 0.001 to 0.08, demonstrating that the system adapts the significance level according to study characteristics rather than applying a single fixed threshold$\alpha = 0.05$.



Among the 20 studies, 11 received a recommended alpha value lower than 0.05, indicating a stricter decision rule. Three studies received a recommendation equal to 0.05, and six received a recommendation higher than 0.05.

	\includegraphics[width=0.7\textwidth]{Images/Alphabar}
	\caption{Comparison of Recommended $\alpha$ Values Across 20 Studies}
	\label{fig:Alphabar}

In the 11 cases where the recommended alpha was stricter than 0.05, the null hypothesis would not have been rejected under the proposed system. Because these studies later failed to replicate, these cases represent instances in which the system would have avoided potentially misleading rejections.

In the three cases where the recommended alpha matched 0.05, the system’s recommendation was consistent with the conventional threshold.

In the six cases where a more lenient alpha was recommended, the system reflected a higher tolerance for false positives based on study characteristics. A more lenient alpha does not imply that the original result was correct; rather, it reflects the contextual balance between Type I and Type II errors within that specific study.

\subsection{Overall Result}

Compared with the conventional use of $\alpha = 0.05$, the proposed alpha recommendation system provides improved decision support in several failed replication cases. As illustrated in Figure\ref{fig:Alphabar}, the blue bars represent studies for which the system recommends stricter significance levels ($\alpha < 0.05$), suggesting that a lower threshold could help reduce the risk of false positive conclusions. These stricter recommendations appear in more than half of the evaluated studies, indicating that the conventional threshold may often be too lenient for the given study conditions. In three cases, the system retained the conventional threshold ($\alpha = 0.05$), indicating that the standard level was appropriate under those conditions. Some studies received more lenient recommendations ($\alpha > 0.05$); however, these values were generated using default contextual settings because the original studies did not report researcher preferences such as risk tolerance or testing context. Incorporating such information could further refine the recommended alpha values.
